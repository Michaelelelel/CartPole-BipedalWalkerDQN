\section{Theory}
The exposition below follows the treatment in [4], supplemented with the primary DQN and Double DQN papers [1, 2] for the algorithmic details specific to this project.

\subsection{Reinforcement Learning}

Reinforcement Learning is a type of Machine Learning where the agent learns by trial and error to optimize its behaviour for a return heuristic, as opposed to being trained on labeled examples. In Deep RL, the value functions below are approximated by a Deep Neural Network (DNN) — a universal function approximator that also performs automatic feature engineering — rather than a lookup table, which is what lets the methods described here scale to continuous or high-dimensional state spaces.

The return heuristic is calculated over episodes, sequences of interactions between agent and environment. In RL, episodes are modelled by Markov Decision Processes (MDPs); we call specific realizations of episodes trajectories.

\subsubsection{Markov Decision Process (MDP)}

An MDP is a sequence of states $s_i$, actions $a_i$ and rewards $r_i$,
\[
s_1, a_1, r_1, s_2, a_2, r_2, s_3, a_3, r_3, \dots
\]
where each action $a_i$, taken based on state $s_i$, results in a state transition $s_i \rightarrow s_{i+1}$ and reward $r_i$ (both controlled by the environment). This process must satisfy the \textbf{Markov property}: the next state depends only on the current state and action, not on the full history,
\[
P(s_{t+1} | s_t, a_t) = P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, \dots, s_0, a_0).
\]

\subsubsection{Return and Policy}

The return $G_t$ is a function of subsequent rewards, e.g.\ the discounted sum $G_t = \sum_{k=0}^{\infty} \gamma^k r_{t+k+1}$ with $\gamma \in [0, 1]$. The agent's objective is to learn a policy $\pi(a_t|s_t)$ that, for each state, picks the action maximizing the expected return (the environment may be stochastic).

\subsubsection{Value Functions}

To solve an MDP, the agent estimates the "goodness" of a state or state-action pair via value functions, which estimate expected return:
\begin{equation}
V^\pi(s) = \mathbb{E}_\pi [G_t | s_t = s], \qquad Q^\pi(s, a) = \mathbb{E}_\pi [G_t | s_t = s, a_t = a].
\end{equation}
The optimal action-value function $Q^*(s, a) = \max_\pi Q^\pi(s, a)$ ($V^*$ analogously) represents the maximum expected return achievable by any policy; once $Q^*$ is known, the optimal policy follows directly as $\pi^*(s) = \arg\max_a Q^*(s, a)$. For Value-Based methods like DQN, $Q^*$ is the primary target, since it lets the agent rank actions without needing a model of the environment's transitions.

\subsubsection{Bellman Equation}
Value functions decompose recursively via the Bellman equation, which relates the value of a state(-action) pair to the value of its successors. For $Q^*$, the Bellman optimality equation is
\begin{equation}
    Q^*(s, a) = \mathbb{E} [r_{t+1} + \gamma \max_{a'} Q^*(s_{t+1}, a') | s_t=s, a_t=a],
\end{equation}
which is the foundation of the DQN algorithm, as it provides an iterative target for updating an estimate of $Q^*$ from immediate rewards and predicted future value.

\subsubsection{Temporal Difference Learning}
Temporal Difference (TD) learning updates value estimates from incomplete sequences by "bootstrapping" a guess from another guess, rather than waiting for a full episode's return as Monte Carlo methods do. In TD(0), the update target for a state $s$ is the immediate reward plus the discounted value of the next state:
\begin{equation}
    Target = r_{t+1} + \gamma V(s_{t+1}).
\end{equation}
DQN uses TD learning, updating its Q-network at every time step from a single transition $(s, a, r, s')$, which makes it far more sample-efficient than Monte Carlo approaches.

\subsection{Deep Q-Networks (DQN)}

DQN estimates $Q^*$ with the DNN introduced above, $Q(s, a; \theta) \approx Q^*(s, a)$, letting it generalize to states not seen during training rather than requiring a lookup table (LUT) entry for every transition as tabular RL would — making DQN suitable for continuous state spaces. Its action space, however, remains discrete, which can cause jitter and a lack of fine-grained precision in applications like robotics; Section~\ref{sec:implementation} addresses this constraint for BipedalWalker via an explicit discretization scheme.

\subsubsection{DQN Objective and Training}
DQN is a Value-Based, Model-Free algorithm that uses TD learning. The network is trained by minimizing the Mean Squared Error (MSE) of the TD error derived from the Bellman optimality equation:
\begin{equation}
    L_i(\theta_i) = \mathbb{E} \left[ \left( \underbrace{r + \gamma \max_{a'} Q(s', a'; \theta_i^-)}_{\text{TD Target}} - Q(s, a; \theta_i) \right)^2 \right],
\end{equation}
where $\theta_i^-$ represents the parameters of a \textbf{Target Network}, a periodic snapshot of the weights used to stabilize training (see below).

\subsubsection{Action Selection and Exploration}
Because DQN learns from its own experience, it faces the \textit{exploration-exploitation} dilemma: occasionally taking sub-optimal actions is necessary to discover better long-term strategies and learn an accurate action-value function.

\paragraph{Greedy Policy}
A strictly greedy policy always selects the action with the highest estimated value: $a = \arg\max_{a'} Q(s, a')$. This maximizes immediate reward but prevents the agent from discovering better actions.

\paragraph{Epsilon-Greedy ($\epsilon$-greedy)}
The agent selects a random action with probability $\epsilon$ and the greedy action with probability $1-\epsilon$. Often $\epsilon$ is annealed (decreased) over time, moving from high exploration to high exploitation as the agent's confidence in its Q-estimates grows.

\paragraph{Boltzmann (Softmax) Exploration}
Instead of choosing purely at random, Boltzmann exploration assigns each action a probability based on its Q-value using a temperature parameter $\tau$:
\begin{equation}
    P(a|s) = \frac{e^{Q(s, a)/\tau}}{\sum_{a'} e^{Q(s, a')/\tau}}.
\end{equation}
This ensures that actions with higher estimated values are explored more frequently than those with very low values.

\subsubsection{Experience Replay Buffer}
Sequential samples within a trajectory are highly correlated, violating the \textit{i.i.d.} assumption required for stable gradient descent; the agent may also "forget" rare but important experiences once it moves to a different part of the state space. DQN addresses this with an \textbf{Experience Replay Buffer} $\mathcal{D}$: during interaction, transitions $e_t = (s_t, a_t, r_{t+1}, s_{t+1})$ are stored in $\mathcal{D}$, and during training a mini-batch is sampled uniformly at random from $\mathcal{D}$ rather than using only the most recent transition. This breaks temporal correlation in the data and lets the network learn from each experience multiple times, improving sample efficiency and numerical stability.

\subsubsection{Target Networks and the Moving Target Problem}
Using the same parameters $\theta$ for both the current estimate $Q(s, a; \theta)$ and the TD target creates a feedback loop: because
\begin{equation}
    Target = r + \gamma \max_{a'} Q(s', a'; \theta)
\end{equation}
depends on the network's own predictions, any update that reduces the current error also shifts the target for the next state — the "ground truth" the network is trying to learn keeps moving, leading to oscillation or divergence.

DQN solves this \textit{moving target problem} with a second, "frozen" \textbf{Target Network} with its own parameters $\theta^-$, used exclusively to compute the TD target:
\begin{equation}
    y^{DQN} = r + \gamma \max_{a'} Q(s', a'; \theta^-).
\end{equation}
The online network $\theta$ is updated at every training step, while $\theta^-$ is only periodically synced (every $C$ steps, $\theta^- \leftarrow \theta$). Keeping the target fixed between syncs temporarily reduces the problem to supervised learning with a stable objective, decoupling weight updates from target calculation and reducing oscillation.

\subsubsection{Double DQN (DDQN)}
Target networks stabilize training, but standard DQN still suffers a systematic \textit{overestimation bias}: the same network $\theta^-$ both \textit{selects} the greedy action (via $\arg\max$) and \textit{evaluates} it, and because selection uses noisy estimates, actions that are overestimated by chance are more likely to be picked. Double DQN (DDQN) mitigates this by decoupling selection from evaluation — the online network $\theta$ selects the greedy action, while the target network $\theta^-$ evaluates it:
\begin{equation}
    y^{DoubleDQN} = r + \gamma \, Q\big(s', \arg\max_{a'} Q(s', a'; \theta); \theta^-\big).
\end{equation}
Since the two networks are trained on different samples, they are unlikely to overestimate the same action simultaneously, reducing the upward bias — at no cost in additional networks, hyperparameters, or meaningful compute overhead.
